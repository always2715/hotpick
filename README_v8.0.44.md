# STELLATE v8.0.44

기준 버전: v8.0.43  
작성일: 2026-06-25

## 핵심 수정

### 누적 피드 보존

기존 피드 조회는 최신 인덱스에서 현재 TOP20을 찾으면 즉시 반환해, Redis에 남아 있는 과거 공개 콘텐츠를 병합하지 못했습니다.

v8.0.44에서는 다음을 적용했습니다.

- `scope=all` 피드는 현재 TOP20에서 조기 반환하지 않음
- `stellate:v7:content:*` 원본 키와 콘텐츠 인덱스를 함께 검색
- 과거 공개 콘텐츠를 최신순으로 병합하고 slug 중복 제거
- 페이지네이션 전에 누적 목록 전체를 구성
- 현재 TOP20은 상단에 유지하되, 다음 페이지에서 과거 공개 피드 계속 표시
- 관리자 `누적 피드 전체 재구성`으로 과거 공개 콘텐츠 인덱스 복구
- 새 TOP 공개 시 기존 피드 인덱스를 삭제하거나 현재 TOP20으로 교체하지 않음

### 콘텐츠 길이 정책 분리

- 피드 상세 `/feed/[slug]`: 새 콘텐츠 본문 2,000~3,000자
- TOP 클릭 요약 `/{slug}`: 요약·맥락·포인트 합계 최대 1,000자
- 기존 공개 콘텐츠는 길이 정책 변경 때문에 피드에서 숨기지 않음
- 새 콘텐츠는 확인된 Fact Ledger 범위 안에서만 2,000~3,000자를 작성
- 근거가 부족하면 일반론으로 분량을 채우지 않고 후보 검증 실패로 처리

## 변경하지 않은 정책

- 관심도 상대순위 상위 25개 생성 후보 고정
- 성공 후보를 원래 selectionRank 순으로 정렬
- 성공 후보 상위 20개 공개
- 실패 후보 발생 시 21~25위 성공 후보 승격
- 성공 후보 20개 미만이면 기존 공개 TOP 유지
- 썸네일 사전 풀 500개와 TOP 선정 로직 분리
