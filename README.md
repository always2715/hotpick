# STELLATE v8.0.36

STELLATE의 기존 관심도 집계 방식은 유지하면서, **상위 25개 후보를 생성·검증한 뒤 성공한 후보를 원래 순위대로 정렬해 TOP20만 공개**하도록 개선한 버전입니다. 키워드·제목·피드·상세 콘텐츠의 근거 연결과 표현 정확성도 강화했습니다.

## 핵심 변경

- 관심도 상대순위 상위 25개를 고정 생성 풀로 선정
- 후보 25개를 모두 조사·생성·검증
- 실패 항목을 제외하고 `selectionRank` 순서대로 성공 후보 상위 20개 공개
- 1~20위 후보가 실패하면 21~25위의 다음 성공 후보가 자동 승격
- 성공 항목이 20개 미만이면 기존 공개 TOP20 유지
- 공개 순위는 1~20으로 다시 부여하고 원래 순위는 `sourceRank`로 보존
- 조사 후 확인된 대표 엔티티를 키워드로 사용해 잘린 검색 문구 정리
- 제목을 Fact Ledger의 핵심 사건과 Fact ID에 연결
- `입장 발표`, `활동 정보`, `내용 공개` 같은 포괄 제목 차단
- 본문 문장을 주체·행동·대상·시점·수치·조건 기준으로 검증
- 동일 인물·동명이인·다른 사건이 섞이지 않도록 엔티티 일관성 강화
- 피드 요약은 구체 사건·수치·일정 Fact를 우선해 1~2개 문단으로 구성
- 기존 v8.0.35의 Upstash 표시, 관련영상, 출처 중복 제거, 상세 상품성 구조 유지

## 배포 후

기존 v8.0.35 이하 실행은 후보 수와 정책이 달라 재개할 수 없습니다. 관리자에서 기존 실행을 중단하고 새 TOP 작업을 시작해야 합니다. 기존 공개 TOP은 새 성공 후보 20개가 준비될 때까지 유지됩니다.
