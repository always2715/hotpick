# STELLATE v8.0.55

v8.0.55는 운영에서 반복된 `피드 상세 본문 1,000자 미달`, `stage·snapshot·slug alias 누락`, `직전 TOP 이어받기 0개`, 포괄 키워드 중복 진입 문제를 실제 코드 경로 기준으로 수정한 버전입니다.

## 핵심 변경

- 피드 본문이 1,000자 미만이면 같은 검색을 반복하기 전에 기존 Fact Ledger만 이용한 전용 구조 보강을 한 번 수행합니다.
- 검증 Fact가 충분한 결정론적 fallback은 최대 18개 Fact와 3~5개 섹션을 중복 없이 활용합니다.
- 외부 검색·AI 호출 전에 최소 research checkpoint를 publication stage와 run snapshot에 먼저 저장합니다.
- Redis 순간 오류가 있어도 stage와 candidate/slug/stage alias 전체를 최대 3회 쓰고 실제 읽기 검증합니다.
- 마지막 실패 원인을 `lastRootCauseCode`로 보존하고, 시도 한도 도달은 `attemptLimitCode=KEYWORD_ATTEMPT_LIMIT`로 별도 기록합니다.
- 실패한 live 콘텐츠가 있어도 직전 TOP 보충 시 누적 archive의 마지막 정상 공개본을 사용합니다.
- `반도체`, `가스 누출`, `구조팀` 같은 포괄·사건 동작 조각을 TOP25 고정 전에 제외합니다.
- `영주 SK스페셜티서`처럼 기업명 뒤에 잘못 붙은 축약 조사만 제한적으로 정규화합니다.
- 같은 출처 URL에서 갈라진 후보는 동일 사건 후보로 중복 제거합니다.
- 관리자 생성 상태에서 실제 진단 코드와 처리 단계를 표시합니다.
- 상단 `TOP20` 메뉴 왼쪽에 피드 메뉴와 같은 방식의 `📈` 아이콘을 추가했습니다.

## 유지 정책

- 관심도 상대순위 산식과 TOP25 후보 순서는 변경하지 않습니다.
- TOP25는 최대 예비 풀이고 정상 콘텐츠 20개 확보 시 조기 완료합니다.
- 신규 정상 15~19개면 직전 정상 TOP에서 최대 5개만 보충합니다.
- 신규 정상 15개 미만이면 기존 TOP20을 유지합니다.
- 피드 최소 1,000자 기준은 유지하며 Fact 부족을 일반론으로 채우지 않습니다.
- 누적 피드, Unsplash 500개 풀, NEW·유지·상승·하락 정책은 유지합니다.

## 배포

`STELLATE_v8.0.55_FULL.zip` 또는 동일한 전체 런타임 스냅샷인 `STELLATE_v8.0.55_REPLACE_ONLY.zip`의 내부 전체를 저장소 최상위에 덮어쓴 뒤 `APPLY_STELLATE_v8.0.55_CLEANUP.bat`을 실행하십시오.

배포 후 기존 실패 실행을 그대로 재개하지 말고 새 TOP 작업을 시작해야 합니다. 기존 실행의 고정 후보에 v8.0.55에서 제외되는 포괄·조각 키워드가 포함돼 있기 때문입니다.
