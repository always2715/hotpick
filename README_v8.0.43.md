# STELLATE v8.0.43

기준 버전: v8.0.42  
작성일: 2026-06-24

## 핵심 수정

### 피드 목록 표시 최종 보장

운영 페이지에서 피드 총 개수는 존재하지만 실제 목록 카드가 0개인 현상을 기준으로 수정했습니다.

- Redis 피드 Hash/ZSET/콘텐츠 인덱스 결과에 현재 공개 TOP 상세 원본을 항상 병합합니다.
- 병합을 페이지네이션 전에 수행합니다.
- `/api/feed`에서 결과 행이 0개면 현재 TOP 데이터로 실제 피드 행을 재구성합니다.
- `/feed` SSR에서도 동일한 최종 fallback을 적용합니다.
- `total`은 화면에 실제로 렌더링할 수 있는 행 수와 맞춥니다.
- 피드 복구를 위해 TOP을 다시 생성하거나 Redis를 초기화할 필요가 없습니다.

### 썸네일 풀 500개 확장

- 10개 카테고리 × 50개 = 총 500개 슬롯
- 기존 001~010 ID는 유지하고 011~050을 추가
- 기본·넓은 전경·상징적 디테일·야간·미니멀 구도로 다양화
- 기존 100개 풀은 유지한 채 부족한 슬롯만 보충
- 관리자에서 `500개까지 보충` 실행 가능
- TOP 실행 전 풀 상태 자동 점검과 부분 풀 사용 유지
- 최근 20개 중복 방지, 같은 화면 중복 방지, 수동 지정 우선 유지

## 배포 혼합 방지

이전 교체 ZIP에서 일부 런타임 파일만 덮어써 구버전 `topConfig`, 피드 API, 썸네일 카탈로그가 남을 가능성이 있었습니다.
v8.0.43 교체 ZIP은 `components`, `lib`, `pages`, `public`, `scripts`, `styles`, `tests` 전체 런타임을 포함합니다.

## 변경하지 않은 정책

- 관심도 상대순위 상위 25개 생성 후보 고정
- 성공 후보를 원래 selectionRank 순으로 정렬
- 성공 후보 상위 20개 공개
- 1~20위 실패 시 21~25위 성공 후보 승격
- 성공 후보 20개 미만이면 기존 공개 TOP 유지
- 이미지 적합도와 이미지 API 상태는 TOP 점수 및 후보 성공 여부에 사용하지 않음
