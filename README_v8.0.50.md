# STELLATE v8.0.50

## 핵심 수정

누적 피드를 실시간 콘텐츠 상태와 분리한 별도 보존소로 변경했습니다.

- 성공적으로 공개된 피드를 `feed:archive:*`에 누적 저장
- 새 TOP 공개 시 기존 누적 항목을 삭제하거나 현재 TOP20으로 교체하지 않음
- 이후 생성 실패·재작성·stage 오류로 live content가 `failed/private`가 되어도 마지막 정상 공개본 유지
- `/feed` 전체·지난 피드는 누적 보존소를 항상 병합
- 과거 정상 공개 콘텐츠를 보존소로 1회 마이그레이션
- 기존 상세 원본이 실패 상태로 바뀌어도 `/feed/[slug]`는 마지막 정상 공개 archive 본문으로 fallback
- 관리자 `누적 피드 전체 재구성` 실행 시 과거 content 키와 기존 피드 인덱스를 archive에 이관
- 명시적 숨김·휴지통·영구 삭제는 archive에도 반영

TOP25 생성 후보 → 성공 후보 TOP20 공개, 순위변동, 콘텐츠 길이, 공개 문안 정제, Unsplash 500개 풀 정책은 변경하지 않았습니다.

## 적용

`STELLATE_v8.0.50_REPLACE_ONLY.zip`을 저장소 최상위에 덮어쓴 뒤 `APPLY_STELLATE_v8.0.50_CLEANUP.bat`을 실행하고 GitHub Desktop에서 전체 변경·삭제를 커밋해 Push합니다.

배포 후 관리자에서 **누적 피드 전체 재구성**을 한 번 실행하면 기존 공개 콘텐츠를 새 누적 보존소로 이관합니다.
