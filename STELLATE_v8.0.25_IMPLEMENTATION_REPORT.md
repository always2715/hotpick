# STELLATE v8.0.25 구현 보고서

## 문제

v8.0.24는 TOP30을 먼저 확정했지만, 후단 콘텐츠 검증이 현재 이슈 Fact를 사실상 필수로 요구해 정상 키워드도 연속 실패할 수 있었습니다. 사용자의 요구는 키워드 선정 후 시스템이 그 키워드가 무엇인지 조사해 작성하는 것입니다.

## 구현

### 키워드 설명형 피드

- 뉴스 이슈 사실과 프로필 사실을 분리
- 현재 이슈 사실이 없고 프로필 사실이 있으면 `identityMode` 적용
- 인물·제품·작품·기업·지역 등의 정체와 특징으로 C등급 설명형 피드 작성
- 실제 사실 ID와 출처 연결 유지
- 공통 관심 증가 D등급 문구 금지

### TOP 작업 상태

- `review`, `pending` 제거
- 자료 부족은 `retry_wait`로 추가 검색·재작성
- 시도 소진 또는 기술 오류만 `failed`
- 관리자 중단은 `stopped`

### 검색 확장

첫 시도 후 다음 검색어를 추가합니다.

- 키워드 최신
- 키워드 공식
- 키워드 주요 활동
- 키워드 일정
- 키워드 기본정보
- 키워드 프로필

재시도별 캐시 키를 분리해 같은 빈 검색 결과를 반복 재사용하지 않습니다.

### 중단과 반복 방지

- 관리자 중단 플래그 저장
- 콘텐츠 생성 전·후, 배치 시작, 최종 공개 전 중단 확인
- 키워드 최대 시도 제한
- 전체 단계 및 실행시간 제한
- 중단 시 남은 작업을 `stopped`로 기록하고 공개 잠금 해제

## 보존된 기능

- v8.0.20 TOP 상대순위
- v8.0.22 대표 키워드 하나 추출
- v8.0.23 피드 인덱스 원본 재구성
- v8.0.24 피드·요약정보 구조
- 30건 원자적 공개
