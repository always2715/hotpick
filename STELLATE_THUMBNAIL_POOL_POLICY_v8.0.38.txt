[STELLATE 썸네일 이미지 사전 풀 구축 및 자동 선택 정책]

1. 적용 목적

현재처럼 콘텐츠가 생성될 때마다 기사 키워드로 이미지를 검색하면 기사 내용과 관계없는 이미지가 선택되거나, 검색 결과가 없거나, 이미지 품질이 일정하지 않은 문제가 발생할 수 있다.

또한 언론사 기사 이미지, 공식 홈페이지 이미지, YouTube 썸네일은 저작권 및 이용조건 문제가 발생할 수 있으므로 사용하지 않는다.

STELLATE 썸네일은 Unsplash에서 사전에 검수한 이미지 100개를 구축하고, 콘텐츠의 카테고리와 분위기에 가장 적합한 이미지를 자동 선택하는 방식으로 변경한다.


2. 확정 정책

- 썸네일 출처는 Unsplash만 사용한다.
- 언론사 기사 이미지는 사용하지 않는다.
- 기업·기관 공식 홈페이지 이미지는 사용하지 않는다.
- YouTube 썸네일은 사용하지 않는다.
- 콘텐츠별 키워드 실시간 이미지 검색은 기본적으로 수행하지 않는다.
- 사전에 검수한 이미지 풀 안에서만 썸네일을 선택한다.
- 초기 이미지 풀은 총 100개로 구성한다.
- 각 이미지에 내부 관리용 분위기 제목과 분류 태그를 부여한다.
- 콘텐츠 제목의 특정 키워드보다 카테고리와 전체 분위기를 우선해 선택한다.
- 선택된 이미지는 콘텐츠에 이미지 ID로 저장한다.
- TOP 순위 갱신 시 기존 콘텐츠의 이미지를 다시 선택하지 않는다.
- 신규 콘텐츠에만 새로운 이미지를 배정한다.
- 관리자가 수동으로 지정한 이미지는 자동으로 변경하지 않는다.

- 해외, 외교, 국제회의, 무역 → 국제·글로벌
- 영화, 가수, 공연, 방송 → 문화·연예

9. 이미지 반복 방지 규칙

- 동일 이미지는 최근 20개 콘텐츠에서 재사용하지 않는다.
- 동일 TOP 화면 안에서는 같은 이미지를 중복 사용하지 않는다.
- 동일 카테고리 이미지가 연속 3개 이상 나오지 않도록 조정한다.
- 직전 이미지와 구도나 분위기가 지나치게 유사하면 차순위 이미지를 선택한다.
- 누적 사용 횟수가 많은 이미지는 점수를 감점한다.
- 마지막 사용 시점이 가까운 이미지는 점수를 감점한다.
- 사용 가능한 이미지가 부족한 경우에만 최근 사용 제한을 단계적으로 완화한다.
- 동일 사건의 제목이나 순위가 변경돼도 기존 이미지를 유지한다.
- 콘텐츠가 TOP에서 제외됐다가 다시 진입해도 기존 이미지를 유지한다.
- 상세 콘텐츠가 갱신돼도 썸네일은 자동으로 변경하지 않는다.
- 관리자가 직접 지정한 이미지는 반복 방지 로직과 관계없이 유지한다.


10. 콘텐츠 저장 구조

콘텐츠 객체에 다음 항목을 추가한다.

}

핵심 저장값은 thumbnailImageId이다.

이미지 URL만 저장하면 저장소 주소 변경이나 이미지 교체 시 관리가 어렵기 때문에 내부 이미지 ID를 기준으로 관리한다.

관리자가 수동 지정한 경우 다음과 같이 저장한다.

{
  "thumbnailSelectionType": "manual"
}


11. TOP 갱신 시 이미지 처리

- TOP 순위 산정 방식은 기존 정책을 유지한다.
- 기존 TOP 콘텐츠는 기존 썸네일을 유지한다.
- 새로 TOP에 진입한 콘텐츠만 이미지를 신규 배정한다.
- 제목이 일부 수정돼도 동일 사건이면 기존 이미지를 유지한다.
- 상세 콘텐츠가 다시 생성돼도 기존 이미지를 유지한다.
- 이미지가 삭제되거나 비활성화된 경우에만 재선택한다.
- 관리자가 수동 지정한 이미지는 자동 선택보다 우선한다.
- TOP 갱신 실패 시 기존 TOP 목록과 기존 이미지를 그대로 유지한다.
- TOP 순위만 변경된 경우 이미지를 다시 선택하지 않는다.
- 동일 사건 판정 ID가 유지되는 동안 같은 이미지를 사용한다.


12. 이미지 저장 방식

Unsplash 외부 이미지 URL을 화면에서 직접 호출하는 방식보다 선정한 이미지를 서비스 저장소에 보관하는 방식을 권장한다.

처리 흐름

Unsplash 이미지 선정
→ 사진 ID, 작가명, 출처 주소 저장
→ 서비스용 이미지 다운로드
→ 16:9 비율로 크롭 및 리사이즈
→ WebP 형식으로 변환
→ 프로젝트 스토리지 업로드
→ 내부 저장 이미지 URL 사용

권장 이미지 규격

- 비율: 16:9
- 기본 크기: 1280×720
- 모바일 최소 크기: 640×360
- 파일 형식: WebP
- 권장 용량: 이미지당 약 100~250KB
- 파일명: category_number.webp
- 예시: finance_001.webp, ai_004.webp

Unsplash 사진 ID, 작가명, 원본 페이지 주소는 이미지 메타데이터에 계속 보존한다.

14. 오류 및 예외 처리

카테고리 판정 실패

- 기본적으로 사회·생활 또는 일반 중립 카테고리 이미지를 사용한다.
- 단, 소재 태그가 하나 이상 일치하는 이미지를 우선한다.
- 완전히 무관한 이미지는 선택하지 않는다.

적합 이미지 없음

- 동일 카테고리의 중립 이미지를 선택한다.
- 최근 사용 제한을 단계적으로 완화한다.
- 그래도 없으면 전체 공통 중립 이미지를 사용한다.

이미지 로딩 실패

- 같은 카테고리의 차순위 이미지를 선택한다.
- 차순위 이미지도 실패하면 STELLATE 기본 썸네일을 사용한다.
- 이미지 오류를 로그에 기록한다.

이미지 삭제 또는 비활성화

- 해당 이미지를 기존에 사용 중인 콘텐츠는 즉시 변경하지 않는다.
- 실제 파일이 없어 화면 표시가 불가능한 경우에만 차순위 이미지로 재배정한다.

자동 분류 신뢰도가 낮은 경우

- 카테고리 중립 이미지 사용
- 특정 사건을 과도하게 연상시키는 이미지는 제외
- 관리자 검토 대상으로 표시 가능

17. 구현 순서

1단계
Unsplash 이미지 100개를 선정한다.

2단계
각 이미지에 카테고리, 분위기 제목, 분위기 태그, 소재 태그, 톤, 사용 가능 콘텐츠, 사용 금지 콘텐츠를 부여한다.

3단계
이미지를 16:9 WebP로 가공해 서비스 저장소에 업로드한다.

4단계
이미지 메타데이터 JSON 또는 DB 테이블을 생성한다.

5단계
콘텐츠 생성 완료 후 카테고리와 분위기를 판정하는 로직을 추가한다.

6단계
이미지 적합도 점수 계산 로직을 구현한다.

7단계
최근 20개 중복 방지와 사용 횟수 감점 로직을 적용한다.

8단계
선택된 thumbnailImageId를 콘텐츠에 고정 저장한다.

9단계
기존 콘텐츠는 이미지 유지, 신규 콘텐츠만 신규 배정하도록 TOP 갱신 로직과 연결한다.

10단계
관리자 이미지 관리 및 수동 교체 기능을 추가한다.

11단계
이미지 로딩 실패와 적합 이미지 없음에 대한 fallback을 구현한다.

12단계
실제 운영 결과를 확인한 뒤 자주 사용되는 카테고리 이미지 수를 확대한다.

19. 최종 핵심 확정사항

- STELLATE 썸네일은 Unsplash 이미지만 사용한다.
- 공식 사이트, 언론사, YouTube 이미지는 사용하지 않는다.
- 이미지 키워드 실시간 검색 방식은 사용하지 않는다.
- Unsplash 이미지 100개를 사전에 검수해 이미지 풀로 구축한다.
- 이미지 풀은 초기 10개 카테고리별 10개씩 구성한다.
- 각 이미지에는 내부 분위기 제목과 메타데이터를 부여한다.
- 콘텐츠의 특정 키워드보다 카테고리, 분위기, 소재를 기준으로 선택한다.
- 동일 이미지는 최근 20개 콘텐츠에서 재사용하지 않는다.
- 같은 TOP 화면에 동일 이미지가 중복되지 않도록 한다.
- 선택된 이미지는 thumbnailImageId로 콘텐츠에 고정 저장한다.
- 기존 콘텐츠는 TOP 순위나 제목이 바뀌어도 이미지를 유지한다.
- 신규 콘텐츠에만 새로운 이미지를 배정한다.
- 관리자가 수동 지정한 이미지는 자동으로 변경하지 않는다.
- 초기 100개 운영 후 사용량이 높은 카테고리부터 150~200개로 확대한다.
