3. v8.0.46 생성 복구 이력
======================================================================

운영 오류:
[top20_from25_content_incomplete]
상위 25개 후보 중 상세·피드·제목이 0/25개 완료
일부 후보는 피드 상세 본문 최소 1,000자 미충족
다수 후보는 실행별 stage·snapshot·slug alias가 모두 없음
최종 상태 KEYWORD_ATTEMPT_LIMIT

직접 원인:
1) TOP 후보 생성은 fastRefresh 경로라 일반 품질 수정 단계를 건너뜀
2) 1,000자 미만 결과가 최종 검증에서 예외 발생
3) 예외 전에 stage·snapshot·slug alias가 저장되지 않을 수 있었음
4) 자동 재시도 3회가 같은 생성 실패를 반복
5) 기존 stage가 없으므로 finalize 복구도 사용할 근거가 없음

v8.0.46 해결:
1) AI 문안이 1,000자 미만이면 같은 시도 안에서 검증 Fact 기반 fallback 비교
2) 더 충실하고 정확성 검증을 통과한 fallback으로 교체
3) 최종 공개 검증 전에 publication stage와 run snapshot을 먼저 저장
4) 최종 검증 실패 시 저장된 Fact Ledger로 같은 요청 안에서 로컬 복구
5) 복구 성공 시 추가 attempt 없이 generated 처리
6) 자동 시도는 3회 유지
7) 관리자 명시적 재개 시 최대 5회로 확장
8) v8.0.45 실패 실행도 동일 TOP25→TOP20 정책이면 재개 가능

중요:
- Fact가 부족해 1,000자를 만들 수 없으면 정상 실패
- 같은 Fact 반복, 근거 없는 전망, 일반론으로 강제 통과 금지
- 25개를 가짜 콘텐츠로 채우지 않음

4. 신규 진입·유지·순위변동 정책
======================================================================

비교 기준:
- 새 TOP20 공개 직전의 기존 공개 TOP20
- 생성 후보 원본이나 처리 완료 순서와 비교하지 않음

동일 항목 판정 우선순위:
1) stableEventId, eventKey, trendKey 일치
2) slug 일치
3) 조사 후 확정된 topKeyword/keyword 일치
4) identityAliases 또는 keywordAliases 일치

표시:
- 이전 공개 TOP에 없음: NEW
- 동일 항목이며 순위 동일: 유지
- 이전보다 상승: ▲ 상승 폭
- 이전보다 하락: ▼ 하락 폭

저장 필드:
- previousRank
- rankChange
- movementStatus: new/up/down/same
- retained
- newEntry
- previousSlug
- rankIdentityScore

원칙:
- 순위변동 계산은 원자적 공개 직전 수행
- 공개 실패 시 기존 TOP과 기존 순위표시 유지
- 21~25위 후보가 승격돼도 직전 공개 TOP에 같은 사건이 있으면 NEW로 오판하지 않음
- 직전 공개 TOP에서 빠졌다가 다시 들어오면 새 진입으로 표시

5. TOP25 → 성공 TOP20 처리 흐름
======================================================================

1. Cron 또는 관리자 TOP 즉시 시작
2. runId 생성
3. 후보 약 90개 수집
4. 스팸·중복·잘린 문장 제거
5. 관심도 상대순위 계산
6. 상위 25개 고정
7. selectionRank 1~25와 candidateId 부여
8. QStash 요청당 후보 1개 처리
9. 독립 조사
10. 조사 후 대표 엔티티 확정
11. Fact Ledger 구성
12. 상세·피드·제목 생성
13. 1,000자 미만이면 검증 Fact 기반 동일 시도 복구
14. 최종 검증 전 stage·snapshot·slug alias 저장
15. 최종 검증
16. 실패 시 저장 stage의 Fact Ledger로 같은 시도 복구
17. 자동 최대 3회
18. 관리자 재개 시 최대 5회
19. 25개 상태 확정
20. 성공 후보를 selectionRank 오름차순 정렬
21. 앞의 20개 선택
22. 기존 공개 TOP과 순위변동 계산
23. 공개 rank 1~20 부여
24. 20개 원자적 공개
25. 성공 20개 미만이면 기존 TOP 유지

TOP 후보 선정 점수·순위 계산은 v8.0.46과 v8.0.47에서 변경하지 않았습니다.

6. 콘텐츠 정책
======================================================================

피드 상세 `/feed/{slug}`:
- 공개 최소 1,000자
- 사실과 근거가 충분하면 약 5,000자 권장
- 일반적인 권장 범위 3,500~6,000자
- 5,000자 미달만으로 실패시키지 않음
- 6,000자 초과만으로 실패시키지 않음
- 동일 Fact 반복, 근거 없는 전망·평가·추측 금지

TOP 클릭 요약 `/{slug}`:
- summary, why, points 표시 합계 최대 1,000자

Fact Ledger:
- 주체, 행동, 대상, 시점, 수치·단위, 조건 확인
- 공식·공인·신뢰 언론 우선
- 온라인 반응은 사실 근거로 사용 금지

제목:
- 대표 키워드 · 구체 사건
- titleEvidenceFactIds 저장
- 최신 소식, 현재 상황, 관심 집중 등 포괄 제목 금지
