# STELLATE v8.0.29 — 정확성 우선 콘텐츠 엔진

## 목적

v8.0.28에서 TOP30 목록은 생성됐지만 피드·요약이 실제 근거와 맞지 않거나 일반적인 AI 문장으로 채워지고, 관련 영상이 누락되며, 같은 출처가 반복 표시되던 문제를 수정한 버전입니다.

## 확인된 직접 원인

1. Fact Ledger 저장 시 실제 사실문을 `수치 변화가 확인됐습니다`, `상태 변화가 확인됐습니다` 같은 일반문으로 바꾸고 있었습니다.
2. 피드 요약 생성기에 문자열 대신 `{ text, claimIds }` 객체가 전달되어 `[object Object]` 또는 의미 없는 조합이 생길 수 있었습니다.
3. AI가 작성한 문장이 연결된 Fact ID의 사실과 같은 내용인지, 근거에 없는 수치·날짜·전망이 섞였는지 검사하지 않았습니다.
4. 같은 사건을 다르게 표현한 독립 출처의 문장을 별도 사실로 처리해 교차 확인이 약했습니다.
5. YouTube 검색 범위와 신뢰 채널 필터가 지나치게 좁아 직접 관련 영상도 누락될 수 있었습니다.
6. 연관 뉴스와 자료 출처에서 동일 canonical URL이 중복 표시될 수 있었습니다.

## 핵심 수정

### 1. 실제 사실문 보존

- Fact Ledger version 4 적용
- 실제 확인 문장을 그대로 저장
- 일반화된 대체문 생성 제거
- 출처 제목·URL·매체·발행시각 보존
- UTM 등 추적 파라미터를 제거한 canonical URL 기준 중복 제거

### 2. 정확성 검증 게이트

신규 `lib/contentAccuracy.js`에서 다음을 검사합니다.

- 공개 문장마다 유효한 Fact ID 연결
- 문장과 연결 사실의 의미 일치
- 근거에 없는 수치·날짜 차단
- 근거에 없는 전망·영향·가능성 표현 차단
- `현재 상황`, `수치 변화`, `상태 변화` 같은 일반 Fact 차단
- 충돌 사실은 공개 문안에서 제외

AI 결과가 검사를 통과하지 못하면 폐기하고, 검증된 Fact Ledger 문장만으로 결정론적 fallback을 만듭니다.

### 3. 피드·요약 수정

- 객체를 문자열로 변환하던 `[object Object]` 오류 제거
- 상세의 검증 문장 또는 Fact Ledger 실제 사실문에서만 요약 생성
- 요약에서 새로운 사실이나 전망을 추가하지 않음
- 일반적인 AI 문장으로 분량을 채우지 않음
- 근거가 부족하면 부정확한 문안을 공개하지 않음

### 4. 교차 확인 개선

- 단순 문장 일치가 아니라 사건 유형·핵심어·날짜·수치를 함께 비교
- `장애가 발생했다고 발표했다`와 `접속 장애가 발생했다`처럼 같은 사건을 다르게 쓴 두 출처를 교차 확인
- 서로 다른 지역·수치의 날씨 정보처럼 다른 사실은 병합하지 않음

### 5. 관련 영상

- 검색 범위를 최대 30일로 확대
- 공식·방송사·언론사·기관·브랜드·주제 공식 채널 우선
- 키워드 또는 사건명과 직접 관련된 영상만 유지
- 개인 브이로그·팬 편집·추측 영상 제외
- 최대 2개 표시
- YouTube API 키가 없거나 신뢰 가능한 직접 관련 영상이 없으면 억지로 채우지 않음

### 6. 출처 중복 제거

- 연관 뉴스 내부 중복 제거
- 자료 출처 내부 중복 제거
- 연관 뉴스에 이미 표시된 URL은 자료 출처에서 다시 표시하지 않음
- UTM·fragment가 다른 같은 URL도 한 번만 표시

## 버전

- appVersion: `8.0.29`
- contentVersion: `126`
- trendCacheVersion: `46`
- Redis namespace: 기존 유지
