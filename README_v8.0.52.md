# STELLATE v8.0.52

## 해결한 문제

기존 `maxRunSteps=72`는 후보 25개를 요청당 1개씩 처리하고 자동 최대 3회까지 시도하는 구조보다 작았습니다.

정상 최악 조건은 다음과 같습니다.

- 1차 후보 처리: 최대 25단계
- 자동 추가 조사 2회: 최대 50단계
- 최종 공개: 1단계 이상
- 합계: 약 76단계

따라서 무한 반복이 없어도 72단계에서 정상 실행이 중단될 수 있었습니다.

## v8.0.52 정책

- 자동 실행 기본 단계 예산: 96
- 관리자 명시적 재개 기본 단계 예산: 160
- 작업 상태가 실제로 변하면 단계 예산을 24씩 확장
- 절대 안전 한도: 240
- 동일한 진행 상태가 연속 10회 반복되면 `trend_refresh_stalled_loop`로 중단
- 절대 안전 한도 초과는 `trend_refresh_hard_step_limit`로 중단
- 관리자 재개 시 `stepCount`, 무진행 카운터, 단계 예산을 초기화
- 관리자 재개 시 새로운 120분 실행 시간창 적용
- QStash dedupe key에 `queueGeneration`을 포함해 기존 cursor를 안전하게 다시 등록

## 기존 중단 실행 복구

`trend_refresh_step_limit`으로 중단된 v8.0.51 실행은 v8.0.52 배포 후 관리자에서 **중단 지점부터 재개**할 수 있습니다.

재개 시 이미 완료된 후보와 저장된 stage·snapshot은 다시 생성하지 않고, 미완료 후보 또는 finalize 단계부터 이어집니다.

## 유지 정책

- 관심도 상대순위 상위 25개 생성 풀
- 성공 후보 상위 20개 공개
- 신규 성공 15~19개일 때 이전 TOP 최대 5개 보충
- NEW·유지·상승·하락 계산
- 누적 피드 append-only archive
- 피드 최소 1,000자, 충분한 근거가 있으면 약 5,000자 권장
- TOP 요약 최대 1,000자
- Unsplash 썸네일 풀 500개
