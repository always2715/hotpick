# STELLATE v8.0.56

v8.0.56은 TOP25 후보를 고른 뒤 피드와 보조 콘텐츠를 동시에 생성·검증하던 구조를 **피드 우선 생성 구조**로 변경한 버전입니다.

## 핵심 변경

- 처리 순서를 `TOP25 순위 키워드 확정 → 피드 본문 완성 → 피드 요약 생성 → 피드 내용 기반 제목 확정 → Q&A·인스타 카드 파생`으로 고정했습니다.
- AI 필수 출력에서 요약·제목·Q&A·인스타 카드를 제거하고 피드 본문 섹션만 먼저 생성합니다.
- Q&A나 인스타 카드의 오류가 검증된 피드 전체를 실패시키지 않도록 정확성·근거·저작권 검증 범위를 분리했습니다.
- 1,000자 기준은 본문 단독이 아니라 실제 피드 화면의 최종 공개 텍스트를 기준으로 계산합니다.
  - 메인 제목
  - 핵심 요약과 맥락
  - 주요 포인트
  - 대제목·소제목
  - 본문
  - Q&A
  - 화면 구성상 포함되는 공백과 줄바꿈
- 5,000자는 근거가 충분할 때의 권장 목표이며, 5,000자 미달만으로 실패시키지 않습니다.
- 최종 공개 텍스트가 1,000자 미만일 때만 기존 Fact Ledger를 이용한 1회 보강을 수행합니다.
- 공식·공인 자료의 구조화 Fact를 인물·기관·기업·가상자산 같은 넓은 키워드의 기본 사실 근거로 재사용합니다.
- stage 복구용 결정론적 콘텐츠도 같은 피드 우선 순서를 사용합니다.

## 변경하지 않은 사항

- 피드 구성요건과 화면 섹션 구성
- 관심도 상대순위 산식
- TOP25 최대 예비 풀과 정상 TOP20 공개 정책
- 신규 정상 15~19개 + 이전 TOP 최대 5개 혼합 공개
- 신규 정상 15개 미만이면 기존 TOP 유지
- 누적 피드, Unsplash 500개 풀, NEW·유지·상승·하락 정책

## 배포

`STELLATE_v8.0.56_FULL.zip` 또는 동일한 전체 런타임 스냅샷인 `STELLATE_v8.0.56_REPLACE_ONLY.zip`의 내부 전체를 저장소 최상위에 덮어쓴 뒤 `APPLY_STELLATE_v8.0.56_CLEANUP.bat`을 실행하십시오.

보고된 v8.0.55의 0/20 실행은 이미 3회 시도를 소진했고 contentVersion 139 결과를 사용하므로 재개하지 말고, v8.0.56 배포 후 새 TOP 작업을 시작해야 합니다.
