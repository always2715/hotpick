4. 누적 피드 확인
1) /api/feed?page=1&limit=20&scope=all&sort=latest 호출
2) 현재 TOP20이 첫 페이지에 표시되는지 확인
3) total이 20보다 큰 경우 page=2 또는 offset=20에서 과거 피드가 표시되는지 확인
4) /feed 페이지 하단 다음 페이지에서 누적 피드가 계속 표시되는지 확인
5) 관리자 대시보드의 누적 피드 수가 현재 TOP20보다 큰지 확인

기존 인덱스에 과거 피드가 누락됐다면 관리자에서 `누적 피드 전체 재구성`을 한 번 실행합니다.
이 작업은 Redis에 남아 있는 `content:*` 공개 원본을 찾아 피드 인덱스를 복구하며 AI 재생성을 기본적으로 수행하지 않습니다.

5. 길이 정책 확인
- TOP 클릭 `/{slug}`: 요약·맥락·포인트 합계 최대 1,000자
- 피드 상세 `/feed/{slug}`: contentVersion 134부터 본문 2,000~3,000자
- 기존 누적 피드는 길이가 짧아도 삭제하거나 숨기지 않음
- 기존 현재 TOP을 새 길이로 다시 만들려면 배포 후 새 TOP 작업을 정상 실행

6. 금지 작업

- 누적 피드를 현재 TOP20으로 초기화
