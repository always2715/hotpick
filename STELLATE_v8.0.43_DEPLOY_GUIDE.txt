5. 피드 확인
- /feed를 새로 엽니다.
- 현재 TOP 상세 콘텐츠가 존재하면 피드 인덱스가 비어 있어도 목록 카드가 보여야 합니다.
- 총 개수만 표시되고 카드가 0개면 브라우저 캐시 문제가 아니라 배포 파일 혼합 여부를 먼저 확인합니다.
- /api/version이 8.0.43인지 확인합니다.
- /api/feed?page=1&limit=20 응답의 items 배열과 total을 확인합니다.
- 정상 복구 시 errorCode가 FEED_CURRENT_TOP_MERGED 또는 FEED_PAGE_EMERGENCY_TOP_FALLBACK일 수 있습니다.

피드 복구를 위해 다음 작업은 하지 않습니다.
- Redis FLUSHALL
- 기존 공개 TOP 삭제
- 새 TOP 강제 생성
- 정상 상세 콘텐츠 삭제

6. 500개 썸네일 풀 구축
필수 환경변수:
- UNSPLASH_ACCESS_KEY

관리자 > 썸네일 이미지에서 `500개까지 보충`을 실행합니다.
- 기존 100개 및 기존 콘텐츠 연결은 유지
- 부족한 슬롯만 추가
- 요청 제한이나 검색 결과 부족으로 한 번에 500개가 채워지지 않으면 저장된 부분 풀을 즉시 사용
- 이후 같은 버튼을 다시 실행하면 부족분만 이어서 보충
- 현재 TOP에서 이미지가 비어 있는 콘텐츠도 backfill

실제 사진 500개의 ID와 URL은 운영 UNSPLASH_ACCESS_KEY를 사용해 Upstash Redis에 저장됩니다.
ZIP에는 500개 분류 슬롯과 선택 정책이 포함되며 제3자 계정의 운영 API 결과 자체를 고정 포함하지 않습니다.
