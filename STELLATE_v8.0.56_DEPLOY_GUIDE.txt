STELLATE v8.0.56 배포 및 운영 확인

1. STELLATE_v8.0.56_FULL.zip 또는 STELLATE_v8.0.56_REPLACE_ONLY.zip을 별도 폴더에 압축 해제합니다.
2. 내부 파일·폴더 전체를 hotpick 저장소 최상위에 덮어씁니다.
3. APPLY_STELLATE_v8.0.56_CLEANUP.bat을 실행합니다.
4. GitHub Desktop에서 변경·삭제 전체를 확인합니다.
5. 권장 커밋 메시지: fix: enforce feed-first TOP20 content pipeline v8.0.56
6. Commit to main 후 Push origin 합니다.
7. Vercel 배포가 Ready인지 확인합니다.
8. /api/version에서 다음 값을 확인합니다.
   - appVersion 8.0.56
   - contentVersion 140
   - trendCacheVersion 57
9. 기존 공개 TOP과 Redis 데이터를 삭제하지 않습니다.
10. 관리자에서 누적 피드 전체 재구성을 1회 실행해 archiveCount를 확인합니다.
11. 보고된 v8.0.55의 0/20 실패 실행은 재개하지 말고 새 TOP 작업을 시작합니다.
12. 새 실행에서 아래를 확인합니다.
   - contentPipeline: feed_body_first → feed_derived_summary → feed_derived_title → feed_derived_auxiliary
   - feedPublicCharacterCount가 제목·요약·포인트·대제목/소제목·본문·Q&A·표시 공백을 포함하는지
   - 본문 단독이 1,000자 미만이어도 최종 공개 텍스트가 1,000자 이상이면 통과하는지
   - Q&A·인스타 카드 오류가 피드 전체를 CONTENT_ACCURACY_FAILED로 만들지 않는지
   - 최종 공개 텍스트가 1,000자 미만일 때만 same-Fact 1회 보강이 실행되는지
   - 비트코인·인물·기관 같은 넓은 키워드가 공식 구조화 Fact를 이용하는지
   - 신규 정상 15~19개일 때만 이전 TOP 최대 5개가 보충되는지
13. 신규 정상 15개 미만에서는 carryover 0이 정상 정책입니다. 이 경우 기존 TOP20 전체를 유지합니다.
14. Redis FLUSHALL, 피드 초기화, 기존 TOP 삭제는 금지합니다.
15. npm audit fix --force는 실행하지 않습니다.
