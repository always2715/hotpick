STELLATE v8.0.42 배포 및 운영 확인 가이드

1. STELLATE_v8.0.42_REPLACE_ONLY.zip을 별도 폴더에 압축 해제합니다.
2. 내부 파일·폴더 전체를 hotpick 저장소 최상위에 덮어씁니다.
3. GitHub Desktop에서 변경 파일과 신규 파일을 모두 확인합니다.
4. 다른 프로젝트의 app/ 또는 .next가 다시 들어간 경우 APPLY_STELLATE_v8.0.42_CLEANUP.bat을 실행합니다.
5. 커밋 예: fix: guarantee feed list and auto prepare thumbnails v8.0.42
6. Push 후 Vercel Deployment Ready 확인.
7. /api/version에서 appVersion 8.0.42 확인.
8. /feed를 새로 열어 현재 TOP 콘텐츠가 목록에 표시되는지 확인합니다. TOP 재생성은 피드 표시를 위해 필수 아님.
9. 관리자 > 썸네일 이미지에서 풀 건수를 확인합니다.
10. 풀이 0개이면 UNSPLASH_ACCESS_KEY 환경변수를 확인하고 '100개 풀 구축'을 실행합니다. 이 작업은 현재 TOP의 누락 썸네일까지 적용합니다.
11. 이후 TOP 작업 시작 시에도 풀 자동 점검·보충 및 현재 TOP 누락 이미지 backfill이 수행됩니다.

금지:
- Redis FLUSHALL
- 기존 TOP 삭제
- 피드 전체 초기화
- TOP 선정 점수 변경

주의:
- 이미지 풀 100개는 Unsplash API 응답과 호출 한도에 따라 여러 실행에 걸쳐 채워질 수 있습니다. 확보된 유효 부분 풀은 즉시 사용됩니다.
- UNSPLASH_ACCESS_KEY가 없으면 중립 STELLATE fallback이 표시되며 콘텐츠 생성과 TOP 공개는 중단되지 않습니다.
