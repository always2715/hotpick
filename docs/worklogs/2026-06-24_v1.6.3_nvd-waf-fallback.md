# 2026-06-24 v1.6.3 NVD WAF fallback

## Incident

v1.6.2 NVD enrichment returned HTTP 502 even with batch=1. The response body showed:

- application error: Supabase admin upsert failed for vulnerabilities
- upstream status: HTTP 403
- upstream page: Cloudflare block page for supabase.co

This confirmed that NVD retrieval had succeeded and the failure occurred when the normalized vulnerability row was written to Supabase.

## Root cause

A specific full-row JSON payload triggered Supabase's Cloudflare WAF. Because `published_at` remained null, the same CVE stayed in the candidate set and repeatedly blocked the backfill.

## Fix

- retain bulk upsert for normal rows
- isolate WAF failures per row
- PATCH `published_at` first
- PATCH optional fields in groups
- recursively split blocked groups until the exact blocked field is identified
- leave only blocked fields unchanged
- record WAF fallback, partial count, fields and Ray IDs
- improve PowerShell error body output

## Safety

- PATCH changes only named fields
- KEV, EPSS and CISA metadata are not cleared
- no DB reset
- no schema migration
- no new environment variables
