======================================================================
0. 문서 역할과 우선순위
======================================================================

이 문서는 채팅 한도, 대화 소실, 담당자 변경에 대비한 최신 단일 인수인계 문서입니다.
새 대화에서는 이 파일을 먼저 읽은 뒤 실제 소스와 대조해서 개발을 이어갑니다.

우선순위:
1) 실제 최신 소스코드
2) 이 문서의 현재 확정 정책
3) README_v8.0.43.md
4) 과거 버전 문서

실제 확인하지 않은 운영 결과를 완료라고 표시하지 않습니다.
운영 Upstash Redis, QStash, Unsplash 자격증명이 필요한 검증은 로컬 검증과 구분합니다.

======================================================================
1. 프로젝트 개요
======================================================================

서비스명: STELLATE
기존 저장소명: hotpick
도메인: stellate.co.kr

목적:
- Google Trends, 뉴스, 네이버 DataLab 등 관심 신호에서 상승 키워드 탐색
- 관심도 상대순위 상위 25개를 고정 생성 후보로 선정
- 각 후보를 독립 조사하고 Fact Ledger 기반 콘텐츠 생성
- 검증 성공 후보를 원래 selectionRank 순으로 정렬
- 성공 후보 상위 20개 공개
- 상위 후보 실패 시 21~25위 성공 후보 자동 승격
- 상세 콘텐츠, 피드, 요약, 제목, 연관 뉴스, 관련 영상, 인스타 카드 생성
- 성공 후보가 20개 미만이면 새 결과를 부분 공개하지 않고 기존 정상 TOP 유지

STELLATE는 기사 원문을 조합하거나 단순 재작성하는 사이트가 아닙니다.
확인된 구조화 사실을 바탕으로 독립적인 설명형 콘텐츠를 작성합니다.

3. v8.0.43 핵심 장애 원인과 해결
======================================================================

운영 확인 증상:
- 상세 콘텐츠와 TOP은 존재함
- /feed 상단 총 개수는 표시됨
- 실제 피드 카드 목록은 0개
- 메인 화면에 과거 TOP30 표기가 남음
- 썸네일은 실제 사진 대신 기본 불꽃 또는 fallback만 표시

확인된 원인:
1) 피드 데이터 계층에서 total과 items가 서로 다른 소스에서 계산될 수 있었음
2) 피드 인덱스 복구가 성공해도 페이지네이션 이후 items가 빈 배열이 될 수 있었음
3) v8.0.42 REPLACE_ONLY가 일부 변경 파일 중심이어서 기존 저장소의 구버전 런타임 파일이 남을 수 있었음
4) 결과적으로 새 페이지 코드, 구버전 feed API, 구버전 topConfig, 100개 썸네일 카탈로그가 혼합될 수 있었음
5) 썸네일 규칙은 있었지만 운영 Redis에 실제 Unsplash 풀을 구축하지 않으면 fallback이 표시됨

v8.0.43 해결:
- 피드 인덱스 결과와 현재 TOP 상세 원본을 항상 병합
- 병합을 페이지네이션 전에 수행
- /api/feed 최종 단계에서 현재 TOP 데이터로 emergency row 구성
- /feed SSR 최종 단계에도 같은 fallback 적용
- total을 실제 렌더링 가능한 행과 일치
- REPLACE_ONLY에 전체 런타임 소스 포함
- 썸네일 슬롯을 500개로 확대

5. TOP25 → 성공 TOP20 정책
======================================================================

1) 후보 약 90개 수집
2) 스팸·중복·잘린 문구 제거
3) 관심도 상대순위 계산
4) 상위 25개 고정 생성 풀 확정
5) selectionRank 1~25 저장
6) 후보당 독립 조사·Fact Ledger·콘텐츠 생성
7) 후보당 자동 최대 3회 제한 재시도
8) 25개 상태 확정
9) 성공 후보를 selectionRank 오름차순 정렬
10) 앞의 20개 선택
11) 공개 rank 1~20 재부여
12) 원래 순위는 selectionRank/sourceRank로 보존
13) 원래 21~25위 공개 시 promotedFromReserve=true
14) 성공 후보 20개 이상일 때만 원자적 공개
15) 20개 미만이면 기존 TOP 유지

이번 v8.0.43에서 변경하지 않은 항목:
- 후보 수집
- 관심도 점수
- 상대순위
- 중복 통합
- generationPoolCount=25
- publicTopCount=20
- reserve promotion
- 원자적 공개 조건

이미지 유무·이미지 점수·Unsplash API 상태는 TOP 계산에 사용하지 않습니다.

키워드:

- 잘린 기사 제목 조각, 관형어·서술어 중간 표현, 수치로 시작하는 조각 제외

- TOP 제목: 대표 키워드 · 구체 사건
- Fact Ledger의 현재 사건 Fact 우선
- titleEvidenceFactIds 저장
- 포괄 제목 금지: 최신 소식, 현재 상황, 확인된 변화, 관심 집중 등

본문:
- 3~5개 역할형 섹션
- 주체·행동·대상·시점·수치·조건 명확화
- 동일 Fact 반복 금지
- 기사 원문 재작성 금지
- 근거 없는 전망·원인·결과 추가 금지
- 온라인 반응은 사실 근거로 사용하지 않음

- CRLF
- BOM 없음

- 과거 정리 BAT와 과거 인수인계 TXT 삭제
