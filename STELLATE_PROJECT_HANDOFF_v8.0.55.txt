0. 문서 역할과 우선순위
======================================================================

이 파일은 새 채팅·담당자 변경·대화 소실 시 STELLATE 프로젝트를 이어가기 위한 최신 단일 인수인계 문서입니다.
앞으로 새 버전을 만들 때도 완료한 내용뿐 아니라 운영에서 추가 확인할 항목, 미확정 위험, 다음 버전에서 처리할 작업을 반드시 함께 기록합니다.

우선순위:
1) STELLATE_v8.0.55_FULL.zip 실제 소스
2) 이 인수인계서의 확정 정책
3) README_v8.0.55.md
4) 이전 버전 자료

실제 운영 Upstash Redis·QStash·Anthropic·Naver API에서 확인하지 않은 결과는 완료로 표시하지 않습니다.

1. 프로젝트 개요
======================================================================

서비스명: STELLATE
기존 저장소명: hotpick
도메인: stellate.co.kr
프레임워크: Next.js 14 Pages Router, React 18, Node.js 22.x
저장·작업: Upstash Redis, Upstash QStash
Redis namespace: stellate:v7

목적:
- Google Trends, 뉴스, DataLab 등에서 상승 키워드 탐색
- 관심도 상대순위 상위 정상 후보 25개를 최대 예비 풀로 고정
- 1위부터 순서대로 독립 조사와 Fact Ledger 기반 콘텐츠 생성
- 정상 콘텐츠 20개 확보 시 낮은 순위 미처리 후보 생략
- 성공 후보를 selectionRank 순서로 정렬해 상위 20개 공개
- 신규 성공 15~19개면 직전 정상 TOP에서 부족분 최대 5개 보충
- 신규 성공 15개 미만이거나 보충 후 20개를 만들 수 없으면 기존 TOP20 유지
- 상세 콘텐츠와 피드를 누적 보존

STELLATE는 기사 원문을 재조합하지 않습니다.
확인된 구조화 사실을 바탕으로 독립적인 설명형 콘텐츠를 작성합니다.

3. v8.0.55 수정 대상 운영 로그
======================================================================

운영 증상:
- 후보 25/25, 추가 검색 46회, 실행 단계 73/96, 약 63분
- 신규 정상 콘텐츠 2개, 직전 TOP 이어받기 0개
- 피드 상세 본문 최소 1,000자 미달이 3회 반복
- 일부 후보는 단일 처리 후 stage·snapshot·slug alias가 모두 없음
- 마지막 표시가 모두 KEYWORD_ATTEMPT_LIMIT로 합쳐짐
- 포괄·조각 후보 예: 반도체, 가스 누출, 구조팀, 영주 SK스페셜티서

실제 코드에서 확인한 직접 원인:
1) 1,000자 미달 시 동일한 결정론적 fallback을 다시 호출할 뿐, 같은 Fact를 더 잘 구조화하는 전용 복구가 없었음
2) 기존 fallback이 충분한 Fact Ledger가 있어도 일부 Fact만 사용해 1,000자 아래에서 끝날 수 있었음
3) 외부 조사·AI 호출 전에 최소 stage를 만들지 않아 요청이 중간 종료되면 저장 흔적 전체가 없을 수 있었음
4) stage와 run snapshot 이중 저장이 단발성에 가까워 Redis 순간 오류에서 모든 alias가 남지 않을 수 있었음
5) 마지막 실제 오류를 KEYWORD_ATTEMPT_LIMIT가 덮어써 관리자 진단이 어려웠음
6) 혼합 공개가 getContent(slug,{includePrivate:true})를 사용해 실패/private live 콘텐츠가 있으면 누적 archive fallback을 건너뜀
7) 포괄어와 동일 사건에서 잘려 나온 후보가 TOP25에 여러 개 들어갈 수 있었음

4. v8.0.55 실제 수정 내용
======================================================================

A. 피드 1,000자 미달 복구
- 같은 요청 안에서 기존 Fact Ledger만 이용한 분량 미달 전용 구조 보강을 1회 수행
- 외부 검색을 다시 실행하지 않음
- 새로운 사실, 원인, 영향, 전망, 평가 추가 금지
- 같은 Fact를 문장만 바꿔 반복하는 방식 금지
- 보강 결과도 정확성 검증을 다시 통과해야 채택
- AI 보강이 불가능하거나 더 짧으면 결정론적 fallback과 비교
- 결정론적 fallback은 최대 18개 구체 Fact와 최대 5개 섹션을 중복 없이 활용
- Fact가 실제로 부족하면 1,000자를 억지로 채우지 않고 정상 실패

B. durable research checkpoint
- 외부 검색·AI 호출 전에 최소 checkpoint 저장
- 저장 위치:
  1) publication stage key
  2) 실행별 run snapshot hash
- 필수 alias:
  1) candidateId
  2) slug:{slug}
  3) stage:{stageId}
- Redis 쓰기와 읽기 검증을 최대 3회 수행
- stage와 모든 alias가 실제로 확인되지 않으면 저장 성공으로 처리하지 않음

C. 실제 오류 원인 보존
- 대표 오류 코드:
  CONTENT_BODY_TOO_SHORT
  RUN_SNAPSHOT_WRITE_FAILED
  RUN_STAGE_NOT_FOUND
  RUN_STAGE_IDENTITY_MISMATCH
  INSUFFICIENT_KEYWORD_EVIDENCE
  CANONICAL_EVENT_UNRESOLVED
  CONTENT_ACCURACY_FAILED
  CONTENT_LOCK_BUSY
  EXTERNAL_RESEARCH_TEMPORARY_FAILURE
  CONTENT_GENERATION_FAILED
- 최종 실제 원인은 errorCode와 lastRootCauseCode에 저장
- 시도 한도 도달은 attemptLimitReached=true와 attemptLimitCode=KEYWORD_ATTEMPT_LIMIT로 별도 저장
- 관리자 콘텐츠 생성 상태에 진단 코드와 candidatePhase 표시

D. 직전 TOP 보충 0개 오류 수정
- 혼합 공개 carryover 조회에서 includePrivate live 조회를 제거
- 정상 공개 조회를 사용해 failed/private live 콘텐츠가 있어도 feed archive의 마지막 정상 공개본으로 fallback
- 관리자 hidden_feed/private/trashed 상태는 계속 제외
- 신규 후보와 eventKey/slug/대표 키워드가 중복되는 이전 TOP은 계속 제외

E. 후보 품질과 동일 사건 중복 제거
- 다음 표현은 대표 엔티티가 아니므로 TOP25 고정 전에 제외:
  반도체
  가스 누출
  구조팀
  구조대
  구조인력
  구조작업
- 영문·숫자가 섞인 긴 기업명 토큰 끝의 잘못 붙은 ‘서’만 제한적으로 제거
  예: 영주 SK스페셜티서 -> 영주 SK스페셜티
- 일반 사람 이름이나 한글 고유명사 끝 글자는 제거하지 않음
- canonical URL이 같은 출처에서 갈라진 후보는 동일 사건 후보로 중복 제거
- UTM, fbclid, gclid와 fragment 차이는 제거한 뒤 URL 비교

F. UI 변경
- 상단 TOP20 메뉴를 `📈 TOP20`으로 표시
- 기존 피드 메뉴 `📋 피드`와 같은 방식의 아이콘 사용

G. 테스트·저장소 정리 안정화
- v8044 회귀 테스트가 Fake Redis 주입을 해제하지 않아 PASS 출력 후 전체 테스트가 멈추던 문제 수정
- 전체 `npm run test:v8`가 마지막 v8.0.55 테스트까지 실행되고 정상 종료되도록 검증
- 저장소 정리기가 과거 버전 README·handoff·release/test 문서를 동적으로 제거
- 최신 v8.0.55 문서와 확정 썸네일 풀 정책 문서만 유지

6. 콘텐츠·제목·피드 확정 정책
======================================================================

피드 상세 /feed/{slug}:
- 최소 공개 기준 1,000자
- 근거가 충분하면 약 5,000자 권장
- 5,000자 미달 자체는 실패 사유가 아님
- 같은 Fact 반복, 일반론, 근거 없는 전망·평가로 분량 채우기 금지
- 확인 사실로 1,000자를 만들 수 없으면 정상 실패

TOP 클릭 요약 /{slug}:
- 핵심 요약·맥락·주요 포인트 합계 최대 1,000자

Fact Ledger:
- 주체, 행동, 대상, 시점, 숫자·단위, 적용 조건 확인
- 공식·공인·신뢰 언론 우선
- 온라인 반응은 사실 근거로 사용 금지

제목:
- 대표 키워드 · 구체 사건
- titleEvidenceFactIds 저장
- 최신 소식, 현재 상황, 관심 집중 같은 포괄 제목 금지

공개 문안:
- 독자가 알아야 할 사건·배경·수치·일정·조건만 표시
- 언론사명·도메인·검증 절차 설명은 본문에서 제거
- 출처는 하단 연관 뉴스·자료 출처 UI에서 제공
