# STELLATE v8.0.36 변경사항

## 목표

기존 TOP20 집계 방식과 콘텐츠 정확성 기준을 유지하면서 다음 두 문제를 해결합니다.

1. 정확히 20개 후보만 생성해 일부가 실패하면 전체 TOP 갱신이 중단되는 문제
2. 검색 문구·제목·본문이 출처의 핵심 사실보다 포괄적이거나 어색하게 작성되는 문제

## TOP25 생성 풀 → 성공 TOP20 공개

### 선정

- 후보 수집과 관심도 상대순위 계산 방식은 유지합니다.
- 상대순위 상위 25개를 한 실행의 고정 생성 풀로 확정합니다.
- 각 후보에 `selectionRank` 1~25를 부여합니다.
- 실행 중 후보를 임의 교체하거나 순위를 다시 계산하지 않습니다.

### 생성·검증

- 25개 후보를 모두 독립 조사합니다.
- 후보당 자동 최대 3회까지 제한적으로 처리합니다.
- 상세, 피드, 제목, Fact Ledger, 공개 준비 상태를 검증합니다.
- 실패 항목은 기록하지만 정상 성공 항목을 함께 폐기하지 않습니다.

### 공개

- 성공한 후보를 `selectionRank` 오름차순으로 정렬합니다.
- 그중 앞의 20개만 공개합니다.
- 공개 `rank`는 1~20으로 다시 부여합니다.
- 원래 후보 순위는 `sourceRank`와 `selectionRank`로 보존합니다.
- 원래 21~25위 후보가 공개되면 `promotedFromReserve=true`로 기록합니다.
- 성공 후보가 20개 미만이면 새 결과를 부분 공개하지 않고 기존 정상 TOP20을 유지합니다.

예:

```text
고정 생성 후보: 1위~25위
실패: 원래 2위, 7위, 18위
성공 후보 정렬: 1, 3, 4, 5, 6, 8 ... 21, 22, 23
공개 TOP20: 성공 후보 중 원래 순위가 가장 높은 20개
```

## 키워드 정확성

- 후보 검색 문구를 그대로 공개 키워드로 쓰지 않습니다.
- 독립 조사 후 Fact Ledger와 출처 제목에서 확인되는 대표 인물·기업·제품·작품·정책명을 우선합니다.
- 잘린 기사 제목, 수치 조각, 관형어 조각은 후보 단계에서 제거합니다.
- 후보 문구를 축약할 때는 해당 대표 엔티티가 출처로 확인된 경우에만 변경합니다.
- 변경 근거를 `keywordVerification`으로 저장합니다.

## 제목 정확성

- 제목은 Fact Ledger의 현재 이슈 Fact를 우선해 생성합니다.
- 공식·확인 상태, 사건 Fact, 날짜·수치의 구체성에 따라 제목 근거 우선순위를 계산합니다.
- 제목에 사용한 근거 Fact ID를 `titleEvidenceFactIds`로 저장합니다.
- 다음과 같은 포괄 제목은 거부합니다.
  - 입장 발표
  - 활동 정보
  - 내용 공개
  - 현재 상황
  - 최근 이슈
  - 확인된 변화
- TOP 제목은 `대표 키워드 · 구체 사건` 형식을 유지합니다.
- 상세 제목은 사건과 확인된 배경을 설명하되 출처에 없는 전망을 추가하지 않습니다.

## 피드·상세 콘텐츠 정확성

- Fact를 주체, 행동, 대상, 시점, 숫자, 조건으로 분해해 문장과 대조합니다.
- `그`, `이것`, `해당`, `관련`처럼 대상이 모호한 표현으로 문단을 시작하지 않습니다.
- 동명이인, 다른 기업, 다른 경기·작품·정책을 한 콘텐츠에 섞지 않습니다.
- 사건의 발표·시작·종료·예정 시점을 시간 순서에 맞게 씁니다.
- 수치와 단위는 Fact Ledger에 있는 형태를 보존합니다.
- 같은 Fact를 여러 문단에서 표현만 바꿔 반복하지 않습니다.
- 근거 없는 평가, 흥행 전망, 시장 반응, 원인·결과 연결을 추가하지 않습니다.
- 독자가 확인할 내용은 예매일, 신청일, 출시일, 공식 일정, 적용 조건처럼 출처에 있는 정보만 사용합니다.

## 피드 요약

- 구체적인 사건·날짜·수치·행동이 있는 Fact를 우선 정렬합니다.
- 일반론과 배경 설명만으로 요약을 시작하지 않습니다.
- 서로 직접 연결되는 Fact를 최대 1~2개 문단으로 편집합니다.
- 요약 생성 정책 버전은 `feed_summary_v6_precise_editorial`입니다.

## 유지되는 정책

- 공개 TOP은 항상 20개 원자적 교체
- TOP10 인스타그램 카드 정책
- 관리자 즉시 중단
- stage와 run snapshot 이중 저장
- 피드 인덱스 장애 시 공개 상세 원본 fallback
- Upstash Redis·QStash 소형 인프라 크레딧
- 연관 뉴스 최대 3개, 관련영상 최대 2개
- canonical URL 기준 출처 중복 제거
- 온라인 반응을 사실 근거로 사용하지 않음

## 배포 후 주의

- 기존 v8.0.35 이하 실행은 재개하지 않습니다.
- 배포 후 `/api/version`에서 8.0.36, 132, 51, 25, 20을 확인합니다.
- 관리자에서 이전 실행을 중단한 뒤 새 TOP 작업을 시작합니다.
- 기존 공개 콘텐츠는 새 작업이 성공하기 전까지 유지됩니다.
- Redis `FLUSHALL`이나 기존 공개 TOP 삭제는 필요하지 않습니다.
